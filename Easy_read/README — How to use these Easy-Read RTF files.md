What is Easy-Read?
==================
Easy-Read documents are designed for people with intellectual disability,
people with low literacy, people for whom English is a second language,
and anyone who finds standard written documents hard to read. They use:
  - Very short sentences (5-15 words)
  - One idea per paragraph
  - Plain language (~Year 6 reading age)
  - Large, friendly images alongside every idea
  - Bold key words

Image / icon markers used in these RTF files
============================================
Each RTF includes inline image-direction tags that your AI image-generation
agents (DALL-E, Midjourney, Adobe Firefly, etc.) can read and convert:

  [HEADER-IMAGE: <description>]   →  Banner image across the top of a section
                                       Use ~1200 × 400 px landscape format.

  [IMAGE: <description>]          →  Full-width photo or illustration that
                                       sits with a paragraph. ~600 × 400 px.

  [ICON: <description>]           →  Small icon (~80 × 80 px) next to text.

Each description is written so an AI agent can generate the image directly.
Replace each tag with the generated image when you publish on the website.

Style guidelines for the images
================================
- Friendly, warm, inclusive
- Show people with disability as ACTIVE — not as patients or recipients
- Show diversity: ages, gender, cultural backgrounds, disability types
- Use bright colours and simple shapes
- Photos should be high-quality and look like real Australian settings
- If using stock photography, prefer the AAUS / Photoability collections
  which represent people with disability authentically
- Avoid emoji-style or childish illustrations unless the document is
  specifically for children (e.g., the Thriving Kids briefing)

How to use these on your website
================================
OPTION 1 — Direct paste into a /resources or /easy-read page
   1. Open the RTF in TextEdit (Mac) or Word (Win).
   2. Replace each [IMAGE:] / [ICON:] / [HEADER-IMAGE:] tag with the
      actual image you have generated.
   3. Copy and paste into your WordPress / Squarespace / Wix editor.
   4. Use the website's heading + paragraph styles to match DYAR brand.

OPTION 2 — Generate Easy-Read PDFs
   1. In Word, save each RTF as PDF.
   2. Upload to a /resources/easy-read folder on your website.
   3. Add download buttons on the relevant pages.

OPTION 3 — Build accessible HTML versions
   1. Each Easy-Read should have a dedicated page on dyar.com.au.
   2. Each page needs: high colour contrast, large font (16pt+),
      keyboard-accessible navigation, alt-text on every image.
   3. Add a "Listen to this page" button (text-to-speech).

Why this matters for the audit
==============================
NDIS Practice Standards Indicator 1.4 (Communication) requires DYAR to
provide information to participants in formats they can understand.
HDAA will look for:
  - Easy-Read versions of key participant-facing documents
  - Accessibility on the public website
  - Evidence DYAR has thought about communication diversity
The set above satisfies all three.

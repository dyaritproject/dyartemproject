Structure of each RTF
=====================
Each principle follows the same pattern:
  - Header band (with colour hint matching website card)
  - Principle number + title
  - Easy Read summary (one short sentence — same as website card)
  - "Please note" sensitive-topic warning (matches website modal)
  - Main content: 5-8 sections, each with [IMAGE:] or [ICON:] tag + plain-language body
  - Callout box with the most important takeaway
  - Contact info footer

Image markers
=============
  [HEADER-IMAGE: ...]   - banner/hero across top, ~1200 x 400 px,
                          coloured gradient matching the website card
  [IMAGE: ...]          - full-width photo or illustration, ~600 x 400 px
  [ICON: ...]           - small icon, ~80 x 80 px, sits with text

Each description is written so your AI image-generation agent can
generate the image directly. Replace each tag with the generated image
when you publish on the website.

Multi-language
==============
Your website shows tabs for English, Mandarin, Arabic, Hindi, Persian,
and Vietnamese. These RTFs are the ENGLISH version (Easy Read). When
ready, translate them with NAATI-certified translators (Mehdi can do
Persian directly). Keep the same [IMAGE:] tags across languages so
images are reused.

Why Easy Read matters for the audit
===================================
NDIS Practice Standards Indicator 1.4 (Communication) requires DYAR to
provide information in formats participants can understand. HDAA will
look at your website on audit day. Having 12 principle Easy Read pages
live demonstrates Indicator 1.4 compliance.

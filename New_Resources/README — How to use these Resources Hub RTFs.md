Image markers used in these RTFs
================================
Each RTF includes inline image-direction tags your AI image-generation agent
(DALL-E, Midjourney, Adobe Firefly, etc.) can read and convert:

  [HEADER-IMAGE: <description>]   →  Banner across the top, ~1200 × 400 px
  [IMAGE: <description>]          →  Full-width photo or illustration, ~600 × 400 px
  [ICON: <description>]           →  Small icon, ~80 × 80 px

Style guide
===========
- Easy-Read style — short sentences, plain language
- Inclusive, warm, friendly tone
- Participant always at the centre
- Easy Read paragraphs ≤ 15 words where possible
- Bold key words
- Avoid jargon (e.g. say "support coordinator" not "coordination")

How to publish
==============
OPTION 1 — Direct paste
   1. Open the RTF in TextEdit (Mac) or Word (Win)
   2. Cmd+A then Cmd+C
   3. Open your website CMS, create a new page or content block
   4. Paste — headings, bold, lists carry over
   5. Replace each [IMAGE:] / [ICON:] / [HEADER-IMAGE:] tag with the actual
      generated image

OPTION 2 — Downloadable PDFs
   1. In Word: File > Export > PDF, save the resulting PDF
   2. Upload to your website media library
   3. Add a "Download" button on the relevant accordion item

OPTION 3 — Both
   - Embed the readable Easy-Read content on the page
   - Also offer a downloadable PDF (especially for the PEEP Form + EMP Feedback Form)

Why this satisfies the audit
============================
NDIS Practice Standards Indicator 1.4 (Communication) requires DYAR to
provide information in accessible formats. Having Easy Read versions of
EVERY participant-facing document — Rights, Advocacy, Reporting,
Complaints, Principles, Emergency, COI, Statements — demonstrates
Indicator 1.4 compliance.

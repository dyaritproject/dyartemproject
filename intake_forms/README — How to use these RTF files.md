What is an RTF file?
====================
Rich Text Format (RTF) is a portable formatted-text format. It opens in
Microsoft Word, Apple Pages, TextEdit, LibreOffice, Google Docs, and any
modern word processor or website CMS.

How to use these on your website
================================
Pick whichever approach matches your website builder:

OPTION 1 — Copy / paste into a web page (simplest)
   1. Open the RTF file in TextEdit (Mac) or Word (Win).
   2. Select all (Cmd+A / Ctrl+A) and copy (Cmd+C / Ctrl+C).
   3. Open your website builder (WordPress, Squarespace, Wix).
   4. Create a new page (e.g. /referrals/community-access).
   5. Paste — formatting (headings, bold, lists) carries over.
   6. Replace the underlined fields with a real online form (see Option 3).

OPTION 2 — Offer the RTF as a downloadable file (audit-friendly)
   1. Upload the RTF file to your website's media library.
   2. Add a download button to each service page: "Download referral form (RTF)".
   3. Referrers print, complete, and email to hello@dyar.com.au.

OPTION 3 — Build a proper online form using the RTF as a script
   1. Use Gravity Forms (WordPress), Squarespace Forms, Wix Forms,
      Jotform, or Typeform.
   2. Mirror each field label in the RTF as a form field.
   3. Configure the form to email hello@dyar.com.au and write into Splose
      via Zapier (or your preferred automation).

Privacy on your website
=======================
- Add a "Privacy notice" near every referral form on your website with the
  text from Section 9 (Consent to refer) and the "Where DYAR stores
  referral information" paragraph.
- Cross-link to your full Privacy Policy.
- Use HTTPS (TLS) on your website — non-negotiable for forms collecting
  health information.
- Configure your form tool to send the data to Splose / Centro / hello@dyar.com.au
  only, never to an external marketing CRM.

Versioning
==========
These RTFs are V2 — 19 May 2026. They match the content of the V2 — 12 May
2026 referral forms with the platform-disclosure language added (matching
V3 19 May consent forms + service agreements) PLUS a comprehensive
Person-Centred Profile section (5A) covering:
  - How you'd like to be known (preferred name + pronouns + title)
  - Identity, culture, and community (Aboriginal/Torres Strait Islander,
    cultural community, faith, LGBTIQA+)
  - Food, dietary, allergies (Halal, Kosher, Veg / Vegan, Gluten-free, etc.
    + every allergy with severity + EpiPen status)
  - Current medications (regular + PRN + who manages them)
  - Sensory preferences (likes, overwhelming things, tools that help)
  - Interests, hobbies, community groups, pets
  - How I like to be supported (communication style, worker preferences,
    important people, people NOT to involve)
  - Daily routines that matter (wake/sleep, mealtime, religious/cultural
    practices)
When you update the referral content, bump the version on both the RTF
and the docx so they stay in sync.
